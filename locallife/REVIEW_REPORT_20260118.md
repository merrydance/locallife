# 后端总体审查报告（2026-01-18）

> 审查范围：后端服务入口、路由与中间件、全量 API 分组与链路入口，结合已移除接口清单（docs/deprecations.md）。
> 目标输出：用途说明、业务链路（状态机）、生产健壮性、功能完成度、矛盾/冗余/缺失项。

## 1. 项目用途说明
该项目为“本地生活服务平台”后端，覆盖小程序用户端与商户端的核心业务：
- 用户认证、资料、地址、搜索与浏览。
- 商户入驻、进件、员工与店主管理、门店经营配置。
- 菜品/套餐/库存/桌台/包间管理与预订。
- 订单、支付、退款、配送与骑手管理。
- 运营商、平台统计与风控体系。
- 通知与 WebSocket 实时推送、行为埋点与推荐。

## 2. 业务链路流程梳理（状态机）
> 以下为基于接口动作的状态推进视图，用动作描述状态迁移。

### 2.1 订单链路（用户/商户/厨房/配送）
- 用户侧：创建订单 -> 取消/确认送达/催单/换单。
- 商户侧：接单/拒单 -> 出餐完成 -> 完成订单。
- 厨房侧：开始备餐 -> 出餐完成（KDS）。
- 配送侧：抢单 -> 开始取餐 -> 确认取餐 -> 开始配送 -> 确认送达。

### 2.2 预订链路
- 用户侧：创建预订 -> 取消 -> 到店签到 -> 起菜通知。
- 商户侧：确认 -> 完成 / 未到店。
- 辅助：追加菜品 / 改菜（差量）。

### 2.3 骑手入驻与进件
- 申请草稿 -> 更新基础/证件 -> 提交 -> 重置（被拒后）。
- 进件：绑卡 -> 查询进件状态。

### 2.4 商户入驻与进件
- 申请草稿 -> 基础/图片/证件 OCR -> 提交 -> 重置（被拒后）。
- 进件：绑卡 -> 查询进件状态。

### 2.5 运营商入驻与进件
- 申请草稿 -> 基础/区域/证件 OCR -> 提交 -> 重置。
- 进件：绑卡 -> 查询进件状态。

### 2.6 支付与退款链路
- 创建支付订单 -> 支付回调更新 -> 退款单创建 -> 退款回调更新。
- 电商收付通回调含合单支付、退款、分账、进件状态。

### 2.7 风控/信任分
- 索赔提交 -> 审核；食品安全上报；商户熔断；欺诈检测；恢复申请；申诉处理。

## 3. 项目生产健壮性
**已有保障**
- 中间件：CORS、安全头、HSTS（生产）、请求追踪与结构化日志、Prometheus、限流、超时。
- 认证与授权：PASETO Token + RBAC（Casbin）+ 商户员工权限中间件。
- 安全与上传：签名 URL、敏感文件受控下载；可选本地数据加密器。
- WebSocket：Hub + 可选 Redis Pub/Sub，支持队列可靠投递开关。
- 读iness：就绪检查可检测数据库连通。

**潜在风险/改进点**
- 外部依赖（地图/微信）缺少统一熔断与降级策略（当前仅记录日志并返回错误）。
- Readiness 仅检查数据库，未涵盖 Redis、WebSocket 依赖与外部支付可用性。
- Webhook 回调链路虽完整，但对“回调入库成功后后续异步任务失败”的兜底一致性需持续关注。

## 4. 项目功能完成度
**覆盖范围广**：
- 用户侧：认证、地址、搜索、收藏、浏览、购物车、下单、评价、通知。
- 商户侧：入驻、进件、员工/店主、菜品/套餐/库存、桌台/包间、预订、订单、财务、统计、设备。
- 运维侧：运营商管理与统计、平台统计、风控、配送费配置。
- 生态：支付回调、退款回调、收付通回调、行为埋点、推荐。

**完成度结论**：主链路（商户入驻/商品管理/订单支付/配送/通知）齐全；运营商与平台统计闭环完整；风控与信任分具备业务入口与治理动作。

## 5. 技术实现的矛盾/冗余/缺失
**可能的矛盾/重复**
- 权限校验并行存在 Casbin 与 MerchantStaff 中间件，需持续统一策略，避免重复判定或规则不一致。
- 部分功能存在 “application 与 applyment” 两条路径，虽语义不同（申请 vs 进件），但命名相近易混淆，需要文档和前端明确区分。

**潜在冗余**
- 同一路径的商户员工管理使用不同 middleware 分组注册，逻辑正确但维护上容易误判顺序与权限范围。

**可能缺失**
- 依赖侧健康检查维度不足（Redis、WebSocket、外部支付与地图服务）。
- 外部依赖统一降级/熔断策略未形成集中规范与开关（需与运行环境策略结合）。

**硬编码/空实现问题（本轮补充）**
- 新用户创建默认昵称“微信用户”、默认角色“customer”为硬编码（/v1/auth/wechat-login）。
- 商户详情兜底值硬编码：信誉分 850、平均出餐时间 15 分钟（/v1/public/merchants/:id）。
- 云打印测试接口为占位实现，直接返回 501（/v1/merchant/devices/:id/test）。
