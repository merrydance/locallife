# Phase0 压测/容量基线草案

## 目标
- 为核心链路建立最小可执行压测与容量评估基线。
- 覆盖下单、支付、配送、预订等关键路径。

## 覆盖接口（初版）
- 下单：POST /v1/orders
- 支付：POST /v1/payment/orders, POST /v1/payment/notify（回调模拟）
- 配送：POST /v1/delivery/grab, POST /v1/delivery/confirm
- 预订：POST /v1/reservations

## 负载模型（起步值）
- 并发用户：50 / 100 / 200
- 目标 RPS：50 / 100 / 200
- 阶段：预热 2m → 稳态 10m → 冷却 2m

## 通过标准（起步值）
- P95 延迟 < 800ms（下单/支付/配送/预订）
- 错误率 < 1%
- CPU < 70%（平均）
- DB 连接池活跃占比 < 80%

## 容量估算（待补齐）
- 峰值 GMV/订单/配送量估算与峰值 RPS 推导
- 高峰期缓存命中率目标与关键索引验证

## 备注
- 以上为 Phase0 草案，正式压测应结合真实流量模型与生产规格校准。
