# 核心业务逻辑梳理与演进计划（可勾选）

> 目标：围绕“堂食（扫码点餐）为核心、外卖与预约为分支、预约到店后转堂食”的整体逻辑，形成可执行、可回溯的计划，避免跑偏。

---

## 0. 使用方式
- 每完成一步就打勾 ✅
- 每一步都要有明确产出（文档/清单/流程图/结论）
- 若出现争议，先补充“事实证据”（代码位置/流程截图）再决策

---

## 1. 背景与目标
- [ ] 1.1 明确业务目标与成功标准
  - 产出：目标说明（为什么做）+ 成功指标清单（怎样算完成）
- [ ] 1.2 界定范围与边界
  - 产出：范围说明（堂食/外卖/预约）+ 不做事项清单

---

## 2. 现状与基线（只做事实盘点）
- [ ] 2.1 堂食主流程现状盘点
  - 产出：现有流程清单（扫码→开台→加菜→结账）
- [ ] 2.2 预约与转堂食现状盘点
  - 产出：现有流程清单（预约→到店→转堂食）
- [ ] 2.3 外卖流程现状盘点
  - 产出：现有流程清单（下单→支付→制作→配送）
- [ ] 2.4 当前实现的能力/限制
  - 产出：能力清单 + 限制清单（只写事实）

---

## 3. 目标模型（对齐概念）
- [ ] 3.1 统一“堂食会话/账单组”的概念
  - 产出：概念说明（什么是会话/账单组、边界/字段）
- [ ] 3.2 状态与事件口径统一
  - 产出：状态清单（订单/支付/预约）+ 事件清单（支付成功/到店核销/加菜）
- [ ] 3.3 合流规则明确
  - 产出：预约转堂食规则（条件、数据迁移、库存处理）

---

## 4. 差异清单（现状 vs 目标）
- [ ] 4.1 功能差异
  - 产出：差异列表（缺失/重复/不一致）
- [ ] 4.2 流程差异
  - 产出：差异流程图或列表
- [ ] 4.3 数据与状态差异
  - 产出：关键字段/状态不一致清单

---

## 5. 方案与策略（只做方向性选择）
- [ ] 5.1 方案选型
  - 产出：方案对比表（2-3种即可）
- [ ] 5.2 决策结论
  - 产出：选型结论与理由（清晰可追溯）

---

## 6. 实施步骤（可执行任务拆解）
- [ ] 6.1 拆分里程碑
  - 产出：里程碑列表（M1/M2/M3…）
- [ ] 6.2 任务拆解
  - 产出：任务清单（每条任务可执行）
- [ ] 6.3 风险与依赖
  - 产出：风险清单 + 依赖清单

---

## 7. 验收与复盘
- [ ] 7.1 验收标准
  - 产出：验收标准清单
- [ ] 7.2 交付清单
  - 产出：交付物清单（文档/流程图/接口变更说明）
- [ ] 7.3 复盘要点
  - 产出：复盘要点与改进行动
