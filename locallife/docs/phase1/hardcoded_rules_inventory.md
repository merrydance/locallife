# Phase1 硬编码规则盘点（草案）

> 目标：将现有硬编码规则整理为可规则化清单，后续逐步迁移到规则引擎。

## 优先迁移（风控/拒绝类）

### 1) 外卖拒绝服务名单
- 位置：api/order.go
- 规则：外卖下单时命中行为黑名单则拒绝下单
- 触发点：createOrder（外卖）
- 建议规则域：order
- 建议动作：deny
- 迁移进度：已接入规则引擎（受 RULES_ENGINE_ENABLED 控制）

### 2) 堂食/预订风险提示（提醒型）
- 位置：api/order.go（堂食）/ api/table_reservation.go（预订）
- 规则：命中行为黑名单则通知商户端
- 触发点：创建堂食订单/预订成功
- 建议规则域：order / reservation
- 建议动作：alert
- 迁移进度：已接入规则引擎（受 RULES_ENGINE_ENABLED 控制）

### 3) 骑手申请自动审核
- 位置：api/rider_application.go
- 规则：健康证与身份证 OCR/有效期校验；不满足直接拒绝
- 触发点：提交骑手申请
- 建议规则域：claim（或 rider_application）
- 建议动作：allow/deny + reason
- 迁移进度：旁路记录已接入规则引擎（不影响现有逻辑）

## 次级迁移（策略类）

### 4) 会员余额支付场景限制
- 位置：api/order.go
- 规则：商户会员设置中未允许的场景禁止使用余额支付
- 触发点：订单创建/支付
- 建议规则域：order/payment
- 建议动作：deny
- 迁移进度：已接入规则引擎（受 RULES_ENGINE_ENABLED 控制）
	- 种子规则已补齐：允许 + 拒绝

## 说明
- 订单/预订状态校验属于状态机约束，可先保留硬编码，后续再规则化。
- 以上为第一批迁移建议，后续可扩展到退款/分账/索赔等。
