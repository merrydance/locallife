# CARD-04 外卖首页首屏预算与渐进水合收口

状态：未开始

优先级：P0

所属批次：Batch 2

## 问题目标

把外卖首页从“列表先出来，但后续 per-item 水合持续放大”的状态，收口成有明确首屏预算、渐进水合顺序和重入刷新边界的高频入口页。

## 影响范围

- [weapp/miniprogram/pages/takeout/index.ts](weapp/miniprogram/pages/takeout/index.ts)
- [weapp/miniprogram/pages/takeout/index.wxml](weapp/miniprogram/pages/takeout/index.wxml)
- 相关商户、菜品、详情与首页搜索 API 封装

## 已知问题

- 首页除了拉主列表，还存在逐店补充请求和 per-item fan-out。
- 回前台、分页、弱网和重入时，现有水合策略会持续放大首页负担。
- 首页已是高频流量入口，任何预算失控都会立刻放大成用户可感知的慢和抖动。

## 任务内容

- [ ] 梳理首页首屏请求预算、二屏补充请求和后台渐进水合的职责边界。
- [ ] 把必须首屏可见的信息与可延迟补全的信息拆开，避免所有字段都挤在首屏请求阶段。
- [ ] 优先批量化能批量化的字段，不能批量化的改成受控渐进水合。
- [ ] 定义 `onLoad`、分页加载、回前台和重新进入时的刷新策略，避免无条件全量重拉。
- [ ] 检查首页卡片的视觉信息量，避免为了补信息继续增加首屏扫描负担。

## 完成定义

- [ ] 首页主列表先稳定展示，补充信息再按优先级渐进完成。
- [ ] 首屏请求和 per-item fan-out 明显下降。
- [ ] 回前台和重入刷新只更新真正可能过期的真值。

## 验证要求

- [ ] 对比改造前后的首屏请求数和补充请求策略。
- [ ] 人工验证弱网首屏、分页加载、回前台重入三类场景。
- [ ] review 时使用 `.github/standards/weapp/REVIEW_CHECKLIST.md`，重点检查性能与稳定性、视觉信息密度和状态恢复。
