# CARD-10 商户预订页日视图、汇总与主任务重构

状态：未开始

优先级：P1

所属批次：Batch 4

## 问题目标

把商户预订页从“日汇总、备菜、列表、代客建单和状态流转堆在一页里”的重页面，收口成一个主任务明确、日视图清晰、操作顺手的预订运营页。

## 影响范围

- [weapp/miniprogram/pages/merchant/reservations/index.ts](weapp/miniprogram/pages/merchant/reservations/index.ts)
- [weapp/miniprogram/pages/merchant/reservations/index.wxml](weapp/miniprogram/pages/merchant/reservations/index.wxml)

## 已知问题

- 页面同时承担日汇总、备菜摘要、预订列表、代客建单和状态流转，主任务不清。
- 日期切换和数据读取偏重，会放大运营页的高频使用成本。
- 动作区中存在弱强调按钮承担重要任务的问题，视觉层级不稳。

## 任务内容

- [ ] 重新梳理预订页首屏信息架构，明确“今日概览”“待处理预订”“次要补充信息”的层级。
- [ ] 把日汇总、备菜摘要、预订列表和代客建单之间的关系拆清楚，不让它们在同一层级竞争。
- [ ] 收口日期切换、状态筛选、列表读取和失败恢复的交互节奏，避免每次操作都触发重页面体验。
- [ ] 统一主次动作的按钮强调、布局和弹层操作区，不再让小按钮承担主流程。

## 完成定义

- [ ] 用户能在首屏立刻知道“今天最该处理什么预订任务”。
- [ ] 预订页的概览、列表和动作区层级清楚，不再像多个页面拼在一起。
- [ ] 日期切换和预订处理的日常操作更轻、更稳。

## 验证要求

- [ ] 人工验证日期切换、状态筛选、代客建单、确认/完成/爽约等主要路径。
- [ ] 验证已有数据时的刷新失败和重试表现。
- [ ] review 时使用 `.github/standards/weapp/REVIEW_CHECKLIST.md`，重点检查视觉、交互、性能与稳定性。
