# Rider Task Cards

日期：2026-03-28

## 1. 目标

把骑手侧的大问题拆成可以逐张勾选、逐张回归的小卡，避免继续以“整包大改”的方式推进。

### Phase 2：资金、积分、异常域重建

5. card-05-rider-wallet-withdraw-closure.md

### Phase 3：孤岛页治理

6. card-06-rider-orphan-pages-and-legacy-manage-cleanup.md

### Phase 4：统一验证收口

7. card-07-rider-runtime-validation-and-weak-network-regression.md

## 3. 使用规则

1. 每张卡只处理一个明确主题，不跨域吞并。
2. 未完成回归前，不得把卡标记为完成。
3. 对注册页的删除、迁移或下线决策必须在卡内写清楚，不得口头约定。
